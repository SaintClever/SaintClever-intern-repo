1. What are the key roles in an Agile team, and what does each role do?<br>
   **Developers**:
   <br>
   - Builds frontend, backend, and mobile application(s) / platform(s)
   - Runs QA test
   - Delivers the product in increments (incremently)
     <br>

   **QA (Quality Assurance)**:
   <br>
   - Insures app stability by debuging / testing new (old) features
   - Finds, reports and help reduce and reproduce bugs a user would create
   - Write and maintain test cases (manual or automated)
   - Test user usability, edge cases
   - Works closely with developers
   - Ensures stable and reliable releases (Not on Friday)
     <br>

   **Product Manager (PM)**:
   <br>
   - Define product vision / goal(s)
   - Takes in user feedback and presents it to the dev team
   - Creates, and maintains the product backlog
   - Writes and clarify user stories
   - Decides what is built next
   - Accepts or rejects completed work based on requirements
   - Acts as the bridge between stackholders and the dev team
     <br>
